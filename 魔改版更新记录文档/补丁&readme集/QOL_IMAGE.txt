# 📚 立绘渲染高度与缩放传递管道（Pipeline）说明

整个图片渲染系统的尺寸计算，围绕着**“画框（容器）”**与**“画作（内容）”**的对齐关系展开。

## 🎯 核心概念
1. **`MAX_H_PX` (画框高度)**：决定了这段 HTML 在屏幕上**整体占据的物理像素高度**，也是最终反馈给终端需要“空出多少换行符”的依据。
2. **`SCALE` (缩放比例)**：决定了图片文件（如 `im_av_chara.webp`）在屏幕上渲染时的**实际长宽像素 (`IMG_H_PX`)**。
3. **`ypos` (下沉对齐偏移)**：由于 ERA 的 HTML 渲染特性，图片默认是**顶部对齐**的。为了让矮小的图片（或缩小的图片）能在画框内**底部对齐**，必须计算 `ypos = MAX_H_PX - IMG_H_PX`。

---

## 🛤️ 传递管道（自上而下）

### 1. 应用层 (App / UI Layer)
*例如：`QOL_PRINT_STATE_IMAGE`, `PRINT_SINGLE_CHARA_IMAGE`*
* **职责**：发起打印请求，决定是否要锁定画框高度。
* **规则**：
  * **自适应模式**：传参 `MAX_H_PX = -1`, `SCALE = -1`。全权委托给底层按照玩家的设置动态计算。
  * **锁定尺寸模式**：如果你希望两张图放在同一个 UI 里排版绝对整齐，需要用 `GET_DEFAULT_IMAGE_HEIGHT` 算出统一的 `MAX_H_PX`，并**保持 `SCALE` 参数与其同步**（通常是强制传 100）。

### 2. 构建层 (Builder Layer)
*例如：`BUILD_CHARA_IMAGE_BLOCK`*
* **职责**：解析缺省参数（填补 `-1`），并串联上下游。最后向应用层**返回最终的物理高度 (`RESULT`)**。
* **规则**：
  * 如果外部传来 `SCALE = -1`，它会立刻将其解析为 `100 + isModifier`。
  * 如果外部传来 `MAX_H_PX = -1`，它会调用 `GET_DEFAULT_IMAGE_HEIGHT(BASE_IMG, SCALE)` 算出一个严丝合缝的画框高度。
  * **向下传递**：把算得明明白白的 `MAX_H_PX` 和 `SCALE` 喂给合并器。

### 3. 合成层 (Assembler Layer)
*例如：`ASSEMBLE_HTML_IMAGE_LAYERS`*
* **职责**：组装底图和无限数量的特效/衣服/蒙版图层，生成 HTML 字符串。**它只负责画图，绝不改变外部画框大小。**
* **规则**：
  * 内部使用 `GET_DEFAULT_IMAGE_HEIGHT(..., SCALE)` 算出每张图片的真实渲染高度 `IMG_H_PX`。
  * 核心计算：**`Y_OFF = MAX_H_PX - IMG_H_PX`**。
  * *(之前遇到的 BUG，就是这里收到的 `MAX_H_PX` 比较小，而 `SCALE` 是 100 导致算出的 `IMG_H_PX` 比较大，减出来的 `Y_OFF` 是负数，图片直接飘到了上一行盖住文字)*。

### 4. 输出层 (Render Layer)
*例如：`HTML_PRINTL_PX`*
* **职责**：打印 HTML，并填充空白行。
* **规则**：接收构建层返回的 `MAX_H_PX`，除以字体高度（`GETCONFIG("一行の高さ")`），精准输出足够的 `PRINTL`，防止后续文字与图片重叠。

---

## 💡 开发者避坑指南（Golden Rules）

1. **同频共振定律**：
   **`MAX_H_PX` 与 `SCALE` 必须匹配！**
   如果你在调用时硬编码了 `SCALE = 100`，那么你的 `MAX_H_PX` 也必须是用 `SCALE=100` 算出来的。否则就会出现“画框小、画作大”导致突破上边界（盖住按钮），或者“画框大、画作小”导致图片悬空。

2. **万能缺省值 `-1`**：
   当你不想动脑子算高度和缩放时，把 `H_PX` 和 `SCALE` 全部传 `-1`。系统会自动读取当前的底图尺寸、玩家设定的缩放百分比，为你组装出一个完美贴合的 HTML 块。

3. **度量衡统一器 `GET_DEFAULT_IMAGE_HEIGHT`**：
   在这个系统里，所有的像素高度**唯一且绝对合法**的来源只有这个函数。无论你是要计算画框（给界面排版），还是要计算画作（给图片设置高度），都必须通过它计算，切忌手动 `宽 * 比例`，以防产生浮点截断误差导致的 1px 错位。
