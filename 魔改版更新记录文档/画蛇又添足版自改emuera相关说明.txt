修复 DIV 内图片按钮点击失效问题：
	问题描述：当 div 内存在高度超过一行（LineHeight）的图片时，点击图片下半部分无法触发按钮事件，只有点击图片左上角（全角字符范围内）才能正常响应。
	根本原因：TestChildHitbox 使用 O(1) 快速定位算法 `lineIndex = localY / LineHeight`，假设每个元素高度不超过一行。当图片高度为 100px 时，点击 Y=50 位置计算出 lineIndex=2，但图片实际在第 0 行，导致命中测试失败。

	1. 修复 ConsoleDivPart.cs：
		- 删除 O(1) 快速定位逻辑，改用遍历所有行的精确包围盒测试
		- 遍历 div 内所有 children 行，对每行的 buttons 进行倒序遍历
		- 使用 actualRelPointY = rect.Y + (i * LineHeight) 计算每行基准 Y 坐标

	注释：
	控制严格字体回退模式。参数为 1 时开启强制逐字检查，参数为 0 时关闭。

修复 XML_ADDNODE 在匹配多个节点时只插入最后一次的 Bug
	在 .NET 的 XmlDocument 体系中，一个 XmlNode 实例只能拥有一个父节点。在旧代码中，当 `doSetAll` 为真且匹配到多个目标节点时，循环使用的是同一个 newChild 实例。这导致该节点在 DOM 树中不断被移动，最终只有最后一个目标节点成功添加了该子节点/属性。
	修复：
	引入了本地函数 `GetNewChild()`。
	- 对于节点 (Node) 操作，使用 `doc.ImportNode(sourceNodeForCopy, true)` 每次生成一个带有正确 OwnerDocument 的深拷贝克隆节点。
	- 对于属性 (Attribute) 操作，每次循环都调用 `doc.CreateAttribute` 创建全新的属性实例。
	确保在批量插入时，每个目标节点都能获得独立的子节点/属性对象。

EXISTVAR:
==================================
原版：

输入：string
返回：int
注释：

如果存在与 varName 同名的变量 / 常量则返回对应的正整数，否则返回 0。

为整数类型时，返回值 setbit 1
为字符串类型时，返回值 setbit 2
为常量类型时，返回值 setbit 3
为二维数组时，返回值 setbit 4
为三维数组时，返回值 setbit 5

命令 / 行内函数两种写法均有效。
==================================
改造：

输入：string,(int)
返回：int
注释：

如果存在第二输入变量，则将string所输入的变量视为一个具体的存储单元，判断是否存在，若存在返回1，不存在返回0
如果不存在第二输入变量，则按原版执行

命令 / 行内函数两种写法均有效。

例：

ERB代码：
PRINTFORML {EXISTVAR("CFLAG:0:0", 1)}
PRINTFORML {EXISTVAR("CFLAG")}
PRINTFORML {EXISTVAR("CFLAG:0:0")}

终端输出结果：
1
1
0

==================================
GETSOUNDORBGMINFO(新增):
==================================

读取SOUND通道或BGM通道的音频播放状态

输入：int, int(可省略)
返回：int
注释：

第一参数读取SOUND通道，第二参数读取所需的音频信息
音频信息总共有五种：音频总长度、当前播放进度、正在播放中、音频音量、音频速度
若第二参数存在，则1返回音频总长度、2返回当前播放进度、3返回正在播放中、4返回音频音量、5返回音频速度
返回值位于result:0中
若第二参数不存在，则同时返回所有信息，result:0返回音频总长度、result:1返回当前播放进度、result:2返回正在播放中、result:3返回音频音量、result:4返回音频速度

命令 / 行内函数两种写法均有效。

ISPLAYINGSOUND(新增):
==================================

判断SOUND通道是否正在播放

输入：int（可省略）
返回：int
注释：

输入一个参数以检查某个通道是否有音频正在播放，若正在播放，则返回通道编号，否则返回-1
省略参数的情况下将遍历所有通道，直到检测到正在播放的通道时返回通道编号，或者遍历完所有通道后返回-1

命令 / 行内函数两种写法均有效。

==================================
SOUNDCONTROL(新增):
==================================

控制SOUND通道的音频播放情况

输入：  ① int, int
		② int, int, int, int（可省略）
返回：int
注释：

情况①：第一参数为通道编号，第二参数为控制类型
控制类型总共有两种种：0为暂停，1为继续播放，2为停止

情况②：第一参数为通道编号，第二参数为变速操控编号（仅支持3），第三参数为变速倍率，第四参数为变速类型
若第四参数存在，则变速时音调会随速度改变
若不存在，这变速时音调不改变

以上两种情况下，若通道编号不存在，返回-1，若控制类型不存在，返回-2，若控制成功，返回1

命令 / 行内函数两种写法均有效。

符号渲染修复（字位簇解析 + GDI测量渲染统一）：
==================================

符号渲染修复（字位簇解析 + GDI/Skia测量渲染统一）：
	修复两个符号渲染问题：

	问题一：⚰️等组合字符（U+26B0 + U+FE0F）后被插入多余空格
	原因：BuildFallbacks使用foreach(char)遍历，将变体选择符U+FE0F作为独立字符测量宽度，导致排版为已合并的字位簇分配了两个字符的宽度

	问题二：🎭等Emoji与后续文字重叠
	原因：指定render='gdi'时，排版引擎仍使用SkiaSharp的测量宽度安排位置，而GDI+实际绘制宽度更宽，导致后续字符提前开始绘制

	架构原则：剥夺GDI+的隐形Padding，统一用物理宽度测量；GDI和Skia片段在混合渲染时使用相同的渲染起点（均包含ShapePositionShift）。

	1. ConsoleStyledString.cs - BuildFallbacks：
		- 使用StringInfo.GetTextElementEnumerator按字位簇遍历，正确处理组合字符（如⚰️ = U+26B0 + U+FE0F）和代理对（如🎭 = U+1F3AD）
		- 使用Font.ContainsGlyphs(grapheme)替代Font.ContainsGlyph(@char)进行字形检查，兼容非BMP字符
		- 添加hasSurrogate检查，确保含代理对的文本强制进入逐字位簇回退分析
		- 使用char.ConvertToUtf32获取codepoint，通过FontFactory.GetFallbackTypefaceForCodepoint查找回退字体

	2. ConsoleStyledString.cs - BuildGdiFallbacks：
		- 添加TextFormatFlags.NoPadding | TextFormatFlags.NoPrefix标志，剥夺GDI+隐形边距，获取精准物理宽度
		- 简化为单条目测量（GDI内部自行处理字体回退）

	3. ConsoleStyledString.cs - SetWidth：
		- 新增GDI模式分支：当RenderMode为TEXTRENDERER时，使用_gdiTexts的GDI测量宽度进行排版，确保测量与渲染引擎一致

	4. ConsoleStyledString.cs - DrawTo / DrawToBitmap GDI渲染部分：
		- 添加TextFormatFlags.NoPadding | TextFormatFlags.NoPrefix | TextFormatFlags.PreserveGraphicsClipping标志
		- 渲染起点统一使用point.X（已包含ShapePositionShift），与Skia路径完全对齐
		- Bitmap尺寸增加2像素冗余防止边缘裁切
		- DrawToBitmap同步修正startX，加上ShapePositionShift

	5. FontFactory.cs：
		- 新增GetFallbackTypefaceForCodepoint(int codepoint, string currentFontName)方法：BMP字符委托现有GetFallbackTypefaceForChar，非BMP字符使用SKTypeface.GetGlyph和SKFontManager.MatchCharacter
		- 新增fallbackTypefaceCodepointCache缓存字典
		- ClearFont()中增加对新缓存的清理
